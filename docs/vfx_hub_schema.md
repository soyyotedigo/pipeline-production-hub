# VFX-Hub — Schema & Pipeline Architecture

> Production management system supporting series, features, commercials, and any project type.
> Designed to improve on ShotGrid/Kitsu patterns where they fail.

---

## 1. Design Principles

| Principle                          | Rule                                                                               |
| ---------------------------------- | ---------------------------------------------------------------------------------- |
| **Code is identity**               | Every entity has an immutable `code` used for filesystem paths and API references  |
| **Name is display**                | `name` is free-form, editable anytime, never touches the filesystem                |
| **Code is autogenerated**          | Users never write codes manually. The system generates them from controlled inputs |
| **Paths are resolved, not stored** | Paths are computed from templates + entity codes. Never hardcoded                  |
| **Soft delete only**               | Nothing is deleted, only archived via `archived_at`                                |

---

## 2. Entity Hierarchy

```
Project
├── Episode (optional — only for series/episodic)
│   └── Sequence
│       └── Shot
│           └── Task
│               └── Version
│                   └── ReviewNote
├── Sequence (for features/commercials — no episode parent)
│   └── Shot
│       └── Task
│           └── Version
│               └── ReviewNote
└── Asset
    └── Task
        └── Version
            └── ReviewNote
```

### Project types and hierarchy behavior

| project_type | Hierarchy |
|---|---|
| `series` | Project → Episode → Sequence → Shot |
| `feature` | Project → Sequence → Shot |
| `commercial` | Project → Sequence → Shot |
| `short` | Project → Sequence → Shot |

The Episode layer is **optional** and only active when `project_type = "series"`. All other types skip directly to Sequence.

```python
def generate_shot_code(sequence: Sequence, project: Project) -> str:
    """
    Auto-increments within sequence.
    Last shot in SQ0080 was SQ0080_SH0020 → next is SQ0080_SH0030
```

## 4. Code Autogeneration — Summary Table

| Entity   | Code Format                 | Generated From            | Example         | User Input               |
| -------- | --------------------------- | ------------------------- | --------------- | ------------------------ |
| Project  | `{SANITIZED_NAME}`          | `name` (only at creation) | `COCOA_MELON_3` | name                     |
| Episode  | `EP{production_number:03d}` | `production_number`       | `EP005`         | production_number + name |
| Sequence | `SQ{production_number:04d}` | `production_number`       | `SQ0080`        | production_number + name |
| Shot     | `{seq_code}_SH{auto:04d}0`  | autoincrement × 10        | `SQ0080_SH0030` | name only                |
| Asset    | `{SANITIZED_NAME}`          | `name` (only at creation) | `HERO_ROBOT`    | name + asset_type        |
| Task     | (no code)                   | `entity_id` + `task_type` | —               | task_type pick           |
| Version  | `v{auto:03d}`               | autoincrement             | `v012`          | — (fully automatic)      |

### Code validation rules

```python
import re
import unicodedata

CODE_PATTERN = re.compile(r'^[A-Z][A-Z0-9_]{1,50}$')

def validate_code(code: str) -> str:
    if not CODE_PATTERN.match(code):
        raise ValueError(
            "Code must start with a letter, "
            "contain only A-Z, 0-9, underscore. "
            "2-51 characters."
        )
    return code

def sanitize_name_to_code(name: str) -> str:
    normalized = unicodedata.normalize('NFKD', name)
    ascii_only = normalized.encode('ascii', 'ignore').decode('ascii')
    sanitized = re.sub(r'[^A-Z0-9]', '_', ascii_only.upper())
    sanitized = re.sub(r'_+', '_', sanitized).strip('_')
    validate_code(sanitized)
    return sanitized
```

### 5.2 Template engine

```python
from pathlib import Path

TEMPLATES = {
    # Project
    "project_root":        "{root}/{project}",

    # Episodic paths
    "episode_root":        "{root}/{project}/episodes/{episode}",
    "seq_root_episodic":   "{root}/{project}/episodes/{episode}/sequences/{sequence}",

    # Non-episodic paths
    "seq_root_flat":       "{root}/{project}/sequences/{sequence}",

    # Shots (works for both modes — {seq_root} is resolved first)
    "shot_root":           "{seq_root}/{shot}",
    "shot_work":           "{seq_root}/{shot}/{task}/work/{shot}_{task}_v{version}.{ext}",
    "shot_publish":        "{seq_root}/{shot}/{task}/publish/{shot}_{task}_v{version}.{ext}",

    # Assets
    "asset_root":          "{root}/{project}/assets/{asset_type}/{asset}",
    "asset_work":          "{root}/{project}/assets/{asset_type}/{asset}/{task}/work/{asset}_{task}_v{version}.{ext}",
    "asset_publish":       "{root}/{project}/assets/{asset_type}/{asset}/{task}/publish/{asset}_{task}_v{version}.{ext}",
}


def resolve_path(
    template_name: str,
    context: dict,
    project_type: str = "feature"
) -> Path:
    """
    Resolves a full path from template name and context dict.

    Example context:
    {
        "root": "Z:/projects",
        "project": "COCOA_MELON_3",
        "episode": "EP005",          # only if series
        "sequence": "SQ0080",
        "shot": "SQ0080_SH0030",
        "task": "comp",
        "version": "012",
        "ext": "exr",
        "asset_type": "char",
        "asset": "HERO_ROBOT"
    }
    """

    # Resolve seq_root based on project type
    if project_type == "series":
        seq_root = TEMPLATES["seq_root_episodic"].format(**context)
    else:
        seq_root = TEMPLATES["seq_root_flat"].format(**context)

    context["seq_root"] = seq_root

    template = TEMPLATES[template_name]
    return Path(template.format(**context))


# --- Usage examples ---

# Series shot publish:

    "root": "Z:/projects",
    "project": "COCOA_MELON_3",

    "ext": "exr"

# → Z:\projects\COCOA_MELON_3\episodes\EP005\sequences\SQ0080\SQ0080_SH0030\comp\publish\SQ0080_SH0030_comp_v012.exr
```

## 6. Review & Publish Workflow

### 6.1 States

```
work → publish → review → approved/changes_requested
                              ↓
                        new version (loop)
```

### 6.2 Flow

```
1. Artist works in:
   .../SQ0080_SH0030/comp/work/SQ0080_SH0030_comp_v012.nk

2. Artist publishes → system:
   a. Auto-increments version number
   b. Copies/exports to publish folder
   c. Generates thumbnail + preview (mp4)
   d. Creates Version entity in DB
   e. Sets task status → "rev" (pending review)
   f. Notifies supervisor

3. Supervisor reviews → creates ReviewNote:
   a. Can annotate on specific frames
   b. Changes version status → "app" or "changes_requested"
   c. If approved → task status → "app"
   d. If changes requested → task status → "wip"
   e. Artist gets notification with notes

4. If changes requested → artist creates v013 → loop back to step 2
```

### 6.3 Publish validation

Before a publish is accepted, the system validates:

```python
def validate_publish(version_data: dict, project: Project) -> list[str]:
    errors = []

    # File exists
    if not Path(version_data["main_file"]).exists():
        errors.append("Main file not found")

    # Naming convention matches
    expected = resolve_path("shot_publish", version_data["context"])
    if Path(version_data["main_file"]) != expected:
        errors.append(f"File path doesn't match convention. Expected: {expected}")

    # Frame range (for image sequences)
    if version_data.get("frame_in") and version_data.get("frame_out"):
        shot = get_shot(version_data["shot_id"])
        expected_frames = shot.frame_out - shot.frame_in + 1 + (shot.frame_handles * 2)
        actual_frames = version_data["frame_out"] - version_data["frame_in"] + 1
        if actual_frames != expected_frames:
            errors.append(f"Frame count mismatch. Expected {expected_frames}, got {actual_frames}")

    # Resolution check
    if version_data.get("resolution"):
        specs = project.delivery_specs
        if version_data["resolution"] != specs["resolution"]:
            errors.append(f"Resolution mismatch. Expected {specs['resolution']}")

    return errors  # empty list = publish OK
```

### 7.2 Uniqueness constraints

```sql
-- Project code is globally unique
ALTER TABLE projects ADD CONSTRAINT uq_project_code UNIQUE (code);

-- Episode production_number is unique within project
ALTER TABLE episodes ADD CONSTRAINT uq_episode_prod_num
    UNIQUE (project_id, production_number);

-- Sequence production_number is unique within parent
-- For series: unique within episode
-- For features: unique within project
ALTER TABLE sequences ADD CONSTRAINT uq_sequence_prod_num
    UNIQUE (project_id, episode_id, production_number);

-- Shot code is unique within project (since code includes sequence prefix)
ALTER TABLE shots ADD CONSTRAINT uq_shot_code
    UNIQUE (project_id, code);

-- Asset code is unique within project
ALTER TABLE assets ADD CONSTRAINT uq_asset_code
    UNIQUE (project_id, code);

-- One task per type per entity
ALTER TABLE tasks ADD CONSTRAINT uq_task_per_entity
    UNIQUE (entity_id, task_type);
```

## 8. Where VFX-Hub Improves on ShotGrid/Kitsu

| Problem in ShotGrid/Kitsu                               | VFX-Hub Solution                                          |     |
| ------------------------------------------------------- | --------------------------------------------------------- | --- |
| Code/name are the same field — renaming can break paths | `name` and `code` are separated. Code is immutable        |     |
| User can type any code — typos break conventions        | Code is always autogenerated, never manually entered      |     |
| Episode/sequence hierarchy is rigid or hacky            | `project_type` controls whether episode layer exists      |     |
| Path templates are in YAML/JSON files on disk           | Templates are stored per-project in the database          |     |
| Version numbers can be manually set                     | Version numbers are always auto-incremented               |     |
| Publish validation is external (custom hooks)           | Built-in publish validation pipeline                      |     |
| `production_number` doesn't exist — code IS the number  | `production_number` carries real meaning, code is derived |     |
| Changing a shot's sequence breaks the code              | Shot code includes sequence prefix, both are immutable    |     |

## 9. Migration Path

If starting fresh:

1. Define project with `naming_rules` and `path_templates`
2. Create episodes (if series) with real `production_number`s
3. Create sequences with real `production_number`s
4. Create shots (auto-numbered)
5. Create assets (code from name)
6. System generates full filesystem on first publish

If migrating from ShotGrid/Kitsu:

1. Import existing entities
2. Set `code` from existing ShotGrid code/name
3. Set `production_number` from parsing existing codes
4. Lock all codes as immutable going forward
5. Existing paths remain untouched — new paths follow new convention
